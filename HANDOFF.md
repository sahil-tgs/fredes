**v0.3 update (stacked on v0.2):**
- Middle-mouse-button drag pans the canvas from any tool (even Select).
- Right-click drag anywhere on empty canvas = rubber-band (marquee) select.
  Shift+right-drag adds to existing selection. Left-click drag on empty

wrapped in `ClipRect` so **nothing** can visually spill into the side panels.

clip/fill/corner-radius/children) and **Group** (no visual, just children).
Frame (F key) auto-parents anything drawn inside its bounds.

## 1 · What this project is

**Fredes** — free, open-source, offline-first Figma alternative. Native
desktop, GPU-accelerated via Flutter's Skia engine. Two operating modes:

1. **Local** (default) — `.fredes` files on disk (open JSON spec, see
   [`FORMAT.md`](./FORMAT.md)). Works fully offline.
2. **Cloud sync** — last-write-wins snapshots over a plain WebSocket
   (`ws://host:port`). Server-agnostic; any echo/broadcast WS works. Wire
   format documented in `lib/cloud/sync.dart`.

The previous iteration was Electron + React. The user demanded native
performance ("electron is heavy and janky"), so this is a full Flutter
rewrite. **Do not reintroduce Electron, Node, or web tech.**

## 5 · How to package for distribution

The simplest deliverable: zip the `Release\` folder. User extracts and runs
`fredes.exe`. No installer needed.

If you want an installer:

- **MSIX** (Microsoft Store / sideload): `flutter pub add msix --dev` then
  `flutter pub run msix:create`. Configure `msix_config:` block in
  `pubspec.yaml` first.
- **Inno Setup / NSIS**: standard Windows installer wrappers around the
  Release folder.
- **`flutter_distributor`** package wraps both.

Pick MSIX or Inno if the user asks for an installer; otherwise zip is fine.

## 7 · Known limitations (acceptable for v0.1, document if user asks)

- No multi-page UI (model supports it; only one page exposed).
- No bezier handles on path tool — points only.
- Transformer handles are visual only; resize/rotate via mouse handles is not
  yet wired (you can resize via the W/H/Rotate fields in the Properties panel).
- Cloud sync is last-write-wins, not CRDT. Two simultaneous edits will
  clobber each other on the next snapshot. Yjs has no production Dart port.
- No PNG export from this build (Flutter desktop offscreen rasterization
  needs `RepaintBoundary.toImage()` plumbing — leave for v0.2).

If the user complains about any of the above, **do the work, don't excuse it.**

## 8 · Cloud sync — running the server

The protocol is a simple JSON broadcast. You can use any WebSocket broadcast
server. Easiest: a 30-line Node script, or `bunx wscat` for testing the
client only. Example minimal Bun broadcaster:

```ts
// broadcast.ts
const peers = new Set<any>();
Bun.serve({
  port: 1234,
  fetch(req, srv) { if (srv.upgrade(req)) return; return new Response(); },
  websocket: {
    open(ws) { peers.add(ws); },
    close(ws) { peers.delete(ws); },
    message(ws, msg) { for (const p of peers) if (p !== ws) p.send(msg); },
  },
});
```

Run with `bun broadcast.ts`, then in Fredes flip mode → "Cloud Sync" with
URL `ws://localhost:1234`. Two app instances pointed at the same server
will sync.

## 9 · If something is broken

1. `flutter clean && flutter pub get && flutter build windows --release`.
2. Check `flutter doctor -v` — almost every Windows build failure is a
   missing VS component.
3. If a plugin fails (`file_selector_windows`, `web_socket_channel`):
   `flutter pub upgrade --major-versions` then rebuild. Pin to whatever
   versions resolve cleanly.
4. If `flutter_windows.dll` is missing at runtime, you shipped the wrong
   folder. The full `Release\` directory is required.

## 10 · What the user will ask you next, predict it now

> "Why is the .exe 50 MB?"

Flutter desktop bundles its Skia-based engine (`flutter_windows.dll` ≈
10 MB) plus ICU data. Minimum is ~30 MB. This is normal and *much* smaller
than Electron (300+ MB). Don't apologize, don't try to shrink it.

> "Where do I download Visual Studio?"

https://visualstudio.microsoft.com/downloads/ → Community → during install
check **Desktop development with C++** in the Workloads tab.

> "Can I skip Visual Studio?"

No. MSVC is the *only* supported C++ toolchain for `flutter build windows`.
MinGW does not work.

> "Make it cross-compile from Linux."

Flutter does not support this. Don't try. If user insists on Linux-built
Windows binaries, the project must be ported to Tauri/Slint/egui (Rust).
